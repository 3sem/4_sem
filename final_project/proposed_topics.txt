 
1. Реализовать на С++ парсер путей на графе c КС-ограничениями, использующий GLL как основу (https://github.com/FormalLanguageConstrainedPathQuerying/GLL4Graph).

2. Реализовать GLL-парсер на C++ (https://dl.acm.org/doi/abs/10.1145/3623476.3623521, https://dspace.spbu.ru/bitstream/11701/42730/2/Diplom_Vadim_Abzalov.pdf).

3. Реализовать на С++ парсер путей на графе, использующий грамматики с контекстами как основу (https://arxiv.org/pdf/1405.5598, https://github.com/Yxbcvn410/TwoSidedContextParser).

4. Написать библиотеку, реализующую структуру SPPF (Сжатый лес разбора: https://lark-parser.readthedocs.io/en/latest/_static/sppf/sppf.html) и набор тестов к ней. Убедиться, что бесконечное число путей, извлеченных из тестовых графов, корректно в ней сохраняется.

5. Реализовать многопользовательскую игру крестики-нолики (морской бой, etc) так, чтобы можно было несколько одновременных игр с нескольких компьютеров в одной сети.

6. Реализовать статический анализатор кода на основе поиска паттернов на AST так, чтобы правила поиска паттернов для анализа можно было вводить из конфигурационного файла посредством некоторого языка.

7. Реализовать статический анализатор кода на основе поиска паттернов на LLVM IR так, чтобы правила поиска паттернов для анализа можно было вводить из конфигурационного файла посредством некоторого языка.
