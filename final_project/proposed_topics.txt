 
1. Реализовать на С++ парсер путей на графе c КС-ограничениями, использующий GLL как основу (https://github.com/FormalLanguageConstrainedPathQuerying/GLL4Graph).

2. Реализовать GLL-парсер на C++ (https://dl.acm.org/doi/abs/10.1145/3623476.3623521, https://dspace.spbu.ru/bitstream/11701/42730/2/Diplom_Vadim_Abzalov.pdf).

3. Реализовать на С++ парсер путей на графе, использующий грамматики с контекстами как основу (https://arxiv.org/pdf/1405.5598, https://github.com/Yxbcvn410/TwoSidedContextParser).

4. Реализовать многопользовательскую игру крестики-нолики (морской бой, etc) так, чтобы можно было несколько одновременных игр с нескольких компьютеров в одной сети.

5. Реализовать статический анализатор кода на основе поиска паттернов на AST так, чтобы правила поиска паттернов для анализа можно было вводить из конфигурационного файла посредством некоторого языка.

6. Реализовать статический анализатор кода на основе поиска паттернов на LLVM IR так, чтобы правила поиска паттернов для анализа можно было вводить из конфигурационного файла посредством некоторого языка.
